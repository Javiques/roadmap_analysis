# Handoff: инструмент анализа дорожных карт

## Назначение

Инструмент показывает прогресс дорожных карт из Excel-файла в виде локального HTML-дашборда. Он помогает быстро понять:

- какие задачи идут по плану;
- какие задачи отстают;
- какие задачи перевыполняются;
- какие задачи уже выполнены;
- как факт соотносится с квартальным и годовым планом.

## Основные файлы

- `task-progress-dashboard.html` - основной интерфейс, вся логика чтения Excel, расчетов и графиков находится здесь.
- `dk_new.xlsx` - рабочий Excel-файл для проверки и загрузки в инструмент.
- `dk_main.xlsx` - основной/эталонный файл, не трогать без отдельного явного запроса.
- `dk_new_before_formula_fix.xlsx` - резервная копия после ранних правок формул.
- `vendor/chart.umd.min.js` - локальный Chart.js.
- `vendor/xlsx.full.min.js` - локальный SheetJS для чтения Excel.

## Технологии

- HTML/CSS/vanilla JavaScript.
- Chart.js для графиков.
- SheetJS/XLSX для чтения Excel в браузере.

## Важная структура Excel

Для стандартных листов ожидается структура:

- `A3` - `Квартал`
- `B3` - `Неделя`
- `C3` - `Начало недели`
- `D3` - `Конец недели`
- `E3` - `Квартальный план`
- `F3` - `Годовой план`
- `G3` - `Квартальный прогресс`
- `H3` - `Годовой прогресс`
- `I3` - `Выполнено`
- `J3` - `Комментарий`

Описание задачи читается из первой строки листа.
Ответственный читается из `A2`.

## Текущая логика выполнения

`Выполнено` в Excel считается накопительно с начала года.

Чтобы не вводить в заблуждение при выборе конкретного квартала, инструмент считает отдельную величину:

```text
Выполнено за квартал = выполнено сейчас - выполнено на конец прошлого квартала
```

Пример для `Тираж / Q2`:

```text
Конец Q1: 91
Конец Q2: 212
Выполнено за Q2: 212 - 91 = 121
Квартальный план Q2: 221
Осталось: 100
```

В режиме `Все кварталы` остается накопительное `Выполнено`, потому что сравнение идет с годовым планом.

## Статус плана

Добавлен статус относительно плана:

- `Отстаем`
- `Идем по плану`
- `Перевыполняем`
- `Нет данных`
- `Выполнено`

Логика:

- в режиме `Все кварталы` факт сравнивается с годовым планом на текущую заполненную неделю;
- при выборе конкретного квартала факт за квартал сравнивается с квартальным планом на текущую заполненную неделю;
- выполненная задача не получает статус темпа, для нее основной статус - `Выполнено`.

В карточках задач статус темпа отображается маленьким цветным кружком:

- оранжевый - `Отстаем`;
- зеленый - `Идем по плану`;
- синий - `Перевыполняем`;
- серый - `Нет данных`.

При наведении на кружок показывается только короткий статус.
Текст причины, например `Отставание от плана: 77`, показывается внутри карточки.

## Карточки задач

В режиме `Все задачи` сверху показываются карточки задач.

В карточке:

- название задачи;
- ответственный;
- прогресс;
- цветная полоса прогресса в цвет статуса;
- короткие метрики;
- статусный кружок для незавершенных задач;
- зеленый бейдж `Выполнено` для завершенных задач.

Если у задачи есть описание, карточка переворачивается по клику. На обратной стороне показывается только описание, без повторения названия задачи и без ответственного.

## Графики

Обзор `Все задачи / Все кварталы`:

- график `Прогресс по задачам` был убран;
- оставлен график `Выполнено и план по задачам`;
- в годовом режиме сравниваются `Годовой план` и `Выполнено`.

Обзор `Все задачи / выбранный квартал`:

- показывается `Прогресс по задачам`;
- график `Выполнено и план по задачам` использует `Квартальный план` и `Выполнено за квартал`.

Страница конкретной задачи:

- верхняя карточка `Статус плана` стоит первой;
- при выборе квартала графики используют `Выполнено за квартал`;
- при выборе `Все кварталы` нижний график разделен на 4 блока: `Q1`, `Q2`, `Q3`, `Q4`;
- в квартальных блоках порядок серий: `Квартальный план`, `Годовой план`, `Выполнено за квартал`.

## Важные ограничения

- `dk_main.xlsx` не менять без явного запроса.
- Если `dk_new.xlsx` открыт в Excel, Windows может блокировать запись. Не пытаться обходить это силой.
- Не удалять lock-файл `~$dk_new.xlsx`; он означает, что Excel-файл открыт.
- Листы после первых стандартных могут иметь отличающуюся структуру, поэтому любые массовые правки Excel нужно делать осторожно.

## Проверка

Для быстрой проверки HTML обычно достаточно открыть страницу в headless Chrome и убедиться, что она загружается без ошибки.

Пример проверяемых признаков:

- `<title>Дашборд прогресса задач</title>`
- `Выбрать файл`
- `Загрузите Excel-файл`

После headless-проверок временные профили Chrome нужно удалять из рабочей папки.

## Создание новых дорожных карт

Текущая реализация безопасная: инструмент не пишет напрямую в `dk_new.xlsx`.

Процесс:

1. Пользователь загружает Excel или создает карту без загрузки файла.
2. Нажимает `Новая дорожная карта`.
3. Заполняет мастер.
4. Новая задача появляется в `state.tasks`, карточках, фильтрах, графиках и вкладках.
5. Для сохранения пользователь нажимает `Скачать Excel`.

Если был загружен исходный Excel, экспорт создает копию книги и добавляет в нее новые листы. Если файл не был загружен, экспорт создает новую книгу только с созданными картами.

Ограничение текущей версии: новые листы экспортируются без сложного копирования стилей и формул существующих листов. Это осознанный первый этап, чтобы не ломать исходный Excel.

## Последние изменения интерфейса и внесение прогресса

- После открытия HTML до загрузки Excel скрыты навигация задач, заголовок `Прогресс задач` и фильтр кварталов; в основной области также нет стартовой подсказки. Они появляются после успешной загрузки файла.
- В левой панели сначала показываются задачи и фильтр кварталов, а ниже — выбор Excel, создание новой дорожной карты и экспорт.
- Для загруженной существующей ДК в карточке `Статус плана` есть иконка карандаша в правом верхнем углу. При наведении она показывает подсказку `Внести правки`.
- Карандаш открывает окно внесения прогресса. Пользователь выбирает неделю, вводит `Выполнено` как накопительный факт и при необходимости комментарий. По умолчанию выбрана текущая неделя; если текущая неделя не найдена, выбирается последняя прошедшая.
- Даты в таблицах и выборе недели приводятся к формату `дд.мм.гггг`, включая даты, пришедшие из Excel как строки в американском формате (`месяц/день/год`).
- Клик по затемнённому фону не закрывает ни мастер новой ДК, ни окно внесения прогресса. Закрытие доступно крестиком или кнопкой `Отмена`.
- После сохранения правки сразу отражаются в дашборде. Кнопка экспорта показывает число новых карт и/или правок.

### Экспорт правок прогресса

Инструмент по-прежнему не перезаписывает исходный Excel на диске. `Скачать Excel` создаёт файл `*_updated.xlsx`:

- Для загруженной книги экспорт работает с исходным XLSX-архивом, а не пересобирает все существующие листы через SheetJS. Поэтому оформление, формулы, объединённые ячейки, настройки печати и остальные части прежних ДК сохраняются.
- для количественных ДК записывает факт и комментарий в исходные колонки `Выполнено` / `Комментарий` (обычно I/J);
- для этапных ДК использует фактические колонки этих полей в листе (например, G/H);
- формулы процентов не заменяются;
- дополнительные листы новых ДК добавляются в эту же экспортируемую копию.

Технически экспорт использует `XLSX.CFB` для точечного изменения XLSX-архива. При записи обязательно указан `fileType: "zip"`; без этого библиотека создаёт OLE-контейнер, который Excel не распознаёт как `.xlsx`.

Проверенный сценарий:

1. Загрузить исходный `dk_new.xlsx`.
2. Создать и заполнить новую ДК.
3. Скачать `dk_new_updated.xlsx`.

В результате новая ДК добавляется отдельным листом, а исходные ДК сохраняют корректное отображение. Не использовать ранее сформированные повреждённые `*_updated.xlsx` как исходные файлы для повторной загрузки.
